# Monetization Strategy - TurtlePal

## Revenue Model: Freemium

---

## Pricing Tiers

### Free Tier
**Price**: $0
**Purpose**: User acquisition, product-led growth

**Includes**:
- Core features with limits
- Up to X items/projects
- Community support
- TurtlePal branding

**Limits**:
- X requests/day
- No premium features
- Basic analytics only

---

### Pro Tier
**Price**: $XX/month or $XXX/year (save 2 months)
**Purpose**: Main revenue driver

**Includes**:
- Everything in Free
- Unlimited auth
- Unlimited turtle_encyclopedia
- Unlimited conservation_news
- Priority support
- Advanced analytics
- No branding
- API access

---

### Team/Business Tier
**Price**: $XX/user/month (min 3 users)
**Purpose**: B2B revenue, higher LTV

**Includes**:
- Everything in Pro
- Team collaboration
- Admin controls
- SSO/SAML
- Dedicated support
- Custom integrations

---

### Enterprise (Custom)
**Price**: Contact sales
**Purpose**: High-value contracts

**Includes**:
- Everything in Team
- Custom SLA
- Dedicated account manager
- Custom development
- On-premise option
- Training & onboarding

---

## Pricing Psychology

### Anchor Pricing
- Display yearly savings prominently
- Show Team tier as "Most Popular"
- Use contrast (Free vs Pro) to drive conversions

### Trial Strategy
- 14-day Pro trial for all new users
- Credit card NOT required (lower friction)
- Upgrade prompts during trial
- End-of-trial email sequence

## Monetization Features

### Upgrade Triggers
These actions should prompt upgrade:
- Using auth beyond free limit
- Using turtle_encyclopedia beyond free limit
- Using conservation_news beyond free limit
- Using donation_system beyond free limit
- Using user_profile beyond free limit
- Team collaboration attempt
- Advanced feature access

### Expansion Revenue
Ways to increase ARPU:
- Usage-based pricing for high-volume users
- Add-ons and premium features
- Annual plan incentives
- Team seat expansion
